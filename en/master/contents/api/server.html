Server endpoints¶

protected¶

These routes require an already authenticated user session

GET /api/user/accessToken¶

Requests a userAccessToken for the current user session

Example request:

GET /api/user/accessToken HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript

"someAccessToken"
Status Codes

  • 200 OK – if accessToken was generated

  • 400 Bad Request – if something went wrong

GET /api/user/(string: id)/publicKey¶

Requests the public key of a given user

Example request:

GET /api/user/abc7665df75878a/publicKey HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript

"some public key"
Status Codes

  • 200 OK – if publicKey was found

  • 400 Bad Request – if id was invalid

POST /api/user/publicKey¶

Updates the publicKey of the current session user

Example request:

POST /api/user/publicKey HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript

"some public key"
Form Parameters

  • publicKey – the new public key

Status Codes

  • 200 OK – if publicKey was saved

  • 400 Bad Request – if something went wrong

GET /api/facility/(string: id)/secret¶

Requests the user-specific encrypted secret of a specific facility. The user must be a member of this facility.

Example request:

GET /api/facility/abc123sdasd/secret HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript

"encrypted secret"
Status Codes

  • 200 OK – user is member and secret was sent

  • 400 Bad Request – something went wrong

GET /api/facility/(string: id)/iv¶

Requests the initilization vector of a specific facility. The user must be a member of this facility.

Example request:

GET /api/facility/abc123sdasd/iv HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript

"a very long initialization vector"
Status Codes

  • 200 OK – user is member and iv was sent

  • 400 Bad Request – something went wrong

GET /api/user/session-challenge¶

Requests an encrypted session token to verify the integrity of the clients private key.

Example request:

GET /api/user/session-challenge HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript

"some encrypted token"
Status Codes

  • 200 OK – user is secured and token was sent

  • 400 Bad Request – something went wrong

POST /api/user/session-challenge¶

Sends a session token to verify integrity of users private key

Example request:

POST /api/user/session-challenge HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript
Form Parameters

  • token – the session challenge token

Status Codes

  • 200 OK – if token was valid

  • 400 Bad Request – if something went wrong

keyserver-only¶

These endpoints are only available for the key.box

POST /keyserver/user/validate¶

Validates a given userAccessToken against the corresponding user data.

Example request:

GET /keyserver/user/validate HTTP/1.1
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: text/javascript

"hereBeToken"
Status Codes

  • 200 OK – for a successfully created token

  • 403 Forbidden – for invalid requests (no user, demo user, …)
